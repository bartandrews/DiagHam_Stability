Aix installation

Due to some problems with xlC and autoconf/automake, the installation procedure is a bit different.
First you have to be sure that the global variable $CXX is set to xlC (if not use export CXX=xlC). In order to create correct Makefiles, the configure and Makefile.in files have to be patched. You can find in the scripts directory a perl script called AIXPatch.pl. Run it from the top directory (aka DiagHam). Now you can continue to the standard installation procedure. Beware to use gmake (GNU make) instead of make.

DiagHam has been tested on AIX v5 with xlC v6. The test machine was an IBM power4 with 32 processors and 256Gb.
